# 문제 해결 순서

문제가 생겼을 때 운영자가 확인할 순서입니다. 위에서 아래로 하나씩 확인하세요.

## 예약 안됨

1. 관리자 승인 상태를 확인합니다.
   - 참가자가 `승인 대기`라면 예약 신청이 되지 않습니다.
2. 차단 일정을 확인합니다.
   - 선택한 공간, 날짜, 시간이 차단 일정과 겹치면 예약할 수 없습니다.
3. 운영시간을 확인합니다.
   - 공간 운영시간 밖이면 예약할 수 없습니다.
4. 이미 예약된 시간인지 확인합니다.
   - 다른 신청과 시간이 겹치면 선택할 수 없습니다.
5. 최대 이용시간을 확인합니다.
   - 참가자별 신청 가능 시간을 초과하면 예약할 수 없습니다.
6. 새로고침 후 다시 시도합니다.
   - 화면 정보가 오래되었을 수 있습니다.
7. 계속 실패하면 개발 담당자에게 문의합니다.
   - 이때 참가자 이름, 공간, 날짜, 시간, 화면에 보인 메시지를 함께 전달합니다.

## 공간 안보임

1. `공간 정보 관리`에서 공간이 활성 상태인지 확인합니다.
   - 비활성 상태면 운영에서 제외된 공간입니다.
2. `사용자 노출` 상태인지 확인합니다.
   - `사용자 숨김` 상태면 참가자 화면에 보이지 않습니다.
3. 공간 설정이 저장되었는지 확인합니다.
   - 변경 후 `수정 저장`을 눌렀는지 확인합니다.
4. 참가자 화면을 새로고침합니다.
5. 계속 보이지 않으면 개발 담당자에게 공간명을 전달합니다.

개발 담당자에게 전달할 때는 `is_active`, `is_public_visible` 값을 확인해 달라고 요청하면 됩니다. 운영자는 보통 관리자 화면의 활성 상태와 사용자 노출 상태만 확인하면 충분합니다.

## 관리자 로그인 안됨

1. 관리자 이름을 확인합니다.
2. 전화번호 전체를 확인합니다.
3. 등록된 관리자 계정인지 확인합니다.
4. 브라우저 탭을 닫고 새로 열어 다시 시도합니다.
5. 계속 안 되면 개발 담당자에게 문의합니다.

참고:

- 관리자 로그인 상태는 브라우저 탭에만 저장됩니다.
- 탭을 닫거나 로그아웃하면 다시 로그인해야 합니다.
- 개발 담당자에게는 관리자 등록 상태와 Supabase 연결 상태 확인을 요청하세요.

## 신청이 안 보임

1. 관리자 화면의 `전체 신청 목록`에서 `새로고침`을 누릅니다.
2. 상태 필터가 원하는 값인지 확인합니다.
3. 취소된 신청이면 `취소됨` 상태로 필터를 바꿉니다.
4. 참가자가 신청 완료 메시지를 봤는지 확인합니다.
5. 계속 안 보이면 참가자 이름, 모임명, 공간, 날짜를 개발 담당자에게 전달합니다.

## 취소가 반영되지 않음

1. 참가자 화면에서 `내 신청 확인/수정`의 `새로고침`을 누릅니다.
2. `취소된 신청 보기`를 켭니다.
3. 관리자 화면에서 해당 신청이 `취소됨` 상태인지 확인합니다.
4. 계속 다르게 보이면 개발 담당자에게 신청 정보를 전달합니다.

## 차단 일정이 이상함

1. 차단한 공간이 맞는지 확인합니다.
2. 날짜가 맞는지 확인합니다.
3. 시작시간과 종료시간이 맞는지 확인합니다.
4. 차단 일정이 해제 상태가 아닌지 확인합니다.
5. 참가자 화면에서 날짜를 다시 선택합니다.
6. 계속 이상하면 차단 일정 내용을 개발 담당자에게 전달합니다.

## 운영 종료 후 데이터 확인

운영 시작 전 테스트 신청을 정리해야 한다면 아래를 확인합니다.

1. 보존해야 할 신청이 없는지 확인합니다.
2. `meetings = 0`이 필요한 상황인지 확인합니다.
3. `sessions = 0`이 필요한 상황인지 확인합니다.
4. 개발 담당자에게 [supabase/manual-sql](./supabase/manual-sql)의 정리 파일 실행을 요청합니다.
5. 정리 후 신청 목록이 비어 있는지 확인합니다.
