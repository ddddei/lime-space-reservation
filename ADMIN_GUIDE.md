# 관리자 운영 가이드

## 관리자 로그인

1. 우측 상단 `관리자 모드`를 누릅니다.
2. 관리자 이름과 전체 전화번호를 입력합니다.
3. 로그인 후 관리자 화면으로 이동합니다.

운영 기본 관리자:

- 한필구 / 01033808374

## 참여자 체크리스트

화면 상단의 `참여자 체크리스트`에서 참여자별 예약 승인 상태, Level, 사용 시간을 확인합니다.

- `예약 승인`: 참여자가 예약 신청을 할 수 있는지 결정합니다.
- `예약 가능`: 승인 상태와 잔여 시간을 기준으로 표시됩니다.
- `사용 시간`: 현재 취소되지 않은 신청 기준으로 계산됩니다.

## 예약 관리

`전체 신청 목록`에서 신청자, 모임명, 공간, 날짜, 상태를 필터링합니다.

자주 쓰는 흐름:

1. `새로고침`을 눌러 최신 신청을 불러옵니다.
2. 신청자나 모임명으로 검색합니다.
3. 필요한 경우 `취소`를 눌러 관리자 취소를 진행합니다.

## 신청 취소

1. 취소할 신청 행의 `취소` 버튼을 누릅니다.
2. 확인 모달에서 `신청 취소`를 누릅니다.
3. 성공 메시지를 확인합니다.
4. 참여자 화면에서는 기본 목록에서 숨겨지고, `취소된 신청 보기`에서 `취소됨`으로 보입니다.

## 차단 일정

`관리자 차단 일정` 영역에서 공간, 날짜, 시작시간, 종료시간, 사유를 입력한 뒤 `차단 저장`을 누르면 참가자 예약 화면에 예약 불가 시간으로 반영됩니다.

기존 차단 일정은 `수정`으로 다시 불러와 공간, 날짜, 시간, 사유를 바꿀 수 있습니다. 운영이 끝난 차단 일정은 `해제`를 눌러 비활성 처리합니다. 해제는 삭제가 아니며 운영 이력을 남깁니다.

## 공간 관리

`공간 정보 관리` 영역은 현재 조회 전용입니다. 공간 master 데이터는 운영 중 임의 삭제하지 않습니다.

유지 대상:

- spaces
- space_images
- operating_hours
- admin_blocks

## 운영 체크리스트

- [ ] 운영 전 참가자 26명 count 확인
- [ ] 관리자 10명 count 확인
- [ ] 김나영 참가자 로그인 확인
- [ ] 한필구 관리자 로그인 확인
- [ ] 테스트 신청 생성/관리자 취소 확인
- [ ] QA 종료 후 meetings/sessions 초기화
- [ ] 운영 URL에서 최종 smoke test

## 문제 발생 시 확인 순서

1. 참가자가 로그인되지 않으면 `participants`의 이름/전화번호/활성 상태를 확인합니다.
2. 예약 버튼이 비활성화되면 승인 상태, Level 사용량, 운영시간, 기존 예약, 차단 일정을 확인합니다.
3. 관리자 로그인이 되지 않으면 `admins`의 이름/전화번호/활성 상태를 확인합니다.
4. 관리자 목록이 비어 보이면 `새로고침` 후 `get_admin_participants` RPC 결과를 확인합니다.
5. 신청 취소 후 참여자 화면이 다르면 참여자 화면의 `새로고침`을 누르고, `취소된 신청 보기`를 확인합니다.
