# 관리자 운영 가이드

운영자가 실제 현장에서 사용하는 관리자 화면 매뉴얼입니다. 빠른 시작은 [QUICK_START.md](./QUICK_START.md), 당일 점검은 [OPERATION_CHECKLIST.md](./OPERATION_CHECKLIST.md)를 함께 확인하세요.

## 로그인

1. 우측 상단 `관리자 모드`를 누릅니다.
2. 관리자 이름을 입력합니다.
3. 전화번호 전체를 입력합니다.
4. 로그인 후 관리자 화면으로 이동합니다.

관리자 로그인은 현재 브라우저 탭에만 유지됩니다. 새로고침해도 관리자 화면은 유지되지만, 브라우저 탭을 닫거나 로그아웃하면 다시 로그인해야 합니다.

운영 기본 관리자:

- 한필구 / 01033808374

## 관리자 화면 구성

관리자 화면은 운영 중 자주 보는 순서대로 구성되어 있습니다.

### 참여자 체크리스트

참가자별 예약 승인 상태와 사용 시간을 확인합니다.

- `예약 승인`: 참가자가 예약 신청을 할 수 있는지 결정합니다.
- `승인 완료`: 예약 신청 가능 상태입니다.
- `승인 대기`: 로그인은 가능하더라도 예약 신청은 할 수 없습니다.
- `Level`: 참가자의 운영 Level을 바꿉니다. Level 1은 8시간, Level 2는 24시간까지 예약할 수 있습니다.
- `사용 시간`: 현재 취소되지 않은 신청 기준으로 계산됩니다.

Level 수정:

1. 변경할 참가자의 `Level` 선택 상자를 누릅니다.
2. `Level 1` 또는 `Level 2`를 선택합니다.
3. 저장 중 표시가 사라지고 사용 시간 한도가 바뀌었는지 확인합니다.
4. 참가자에게 로그아웃 후 다시 로그인하도록 안내하면 변경된 Level과 예약 가능 시간이 반영됩니다.

Level 수정도 관리자 이름과 전화번호 검증을 통과한 경우에만 저장됩니다. 저장 실패 메시지가 보이면 관리자 로그인 상태를 확인한 뒤 다시 시도하세요.

### 참가자 추가 / 비활성

신규 인원 합류나 중도 이탈을 관리자 화면에서 바로 처리합니다. 더 이상 Supabase SQL Editor에서 직접 SQL을 실행할 필요가 없습니다(대량 인원을 한 번에 반영해야 하는 운영 오픈 시에는 여전히 `supabase/manual-sql/01_upsert_participants.sql`을 사용합니다).

참가자 추가:

1. `참여자 체크리스트` 상단의 `참가자 추가`를 눌러 폼을 펼칩니다.
2. 이름, 전화번호(`010-0000-0000` 형식), Level, 메모(선택)를 입력합니다.
3. `추가`를 누릅니다. 신규 참가자는 항상 `승인 대기` 상태로 시작하므로, 자료 확인 후 `예약 승인`을 별도로 켜 주세요.
4. 이미 등록된 이름 + 전화번호와 같으면 오류 메시지가 표시됩니다. 예전에 비활성화했던 사람을 같은 이름·전화번호로 다시 추가하면 자동으로 재활성화됩니다.

참가자 비활성화(제외):

1. 제외할 참가자 행의 `비활성` 버튼을 누릅니다.
2. 확인 창에서 다시 한 번 확인하면 처리됩니다.
3. 비활성화는 예약 이력을 지우지 않는 안전한 처리입니다(데이터를 삭제하지 않고 `is_active`만 꺼집니다). 비활성 참가자는 예약 신청 가능 목록에서 즉시 제외됩니다.
4. 기본 화면에는 비활성 참가자가 보이지 않습니다. 확인하려면 `비활성 포함 보기`를 켜세요. 다시 합류시키려면 회색으로 표시된 행의 `재활성` 버튼을 누릅니다.

### 관리자 계정 관리

관리자 계정 10명의 추가/제외도 이제 관리자 화면에서 바로 처리합니다. 더 이상 Supabase SQL Editor에서 `supabase/manual-sql/02_upsert_admins.sql`을 직접 실행할 필요가 없습니다(엑셀 명단을 통째로 반영해야 하는 대규모 개편 시에만 수동 SQL을 사용하세요).

관리자 계정 추가:

1. `관리자 계정` 섹션 상단의 `관리자 계정 추가`를 눌러 폼을 펼칩니다.
2. 이름, 전화번호(`010-0000-0000` 형식), 역할(선택, 비워두면 `manager`로 등록)을 입력합니다.
3. `추가`를 누릅니다.
4. 이미 등록된 이름 + 전화번호와 같으면 오류 메시지가 표시됩니다. 예전에 비활성화했던 관리자를 같은 이름·전화번호로 다시 추가하면 자동으로 재활성화됩니다.

관리자 계정 비활성화(제외):

1. 제외할 관리자 행의 `비활성` 버튼을 누릅니다.
2. 확인 창에서 다시 한 번 확인하면 처리됩니다.
3. 비활성화는 데이터를 삭제하지 않는 안전한 처리입니다(`is_active`만 꺼집니다). 비활성 관리자는 즉시 로그인할 수 없게 됩니다.
4. 기본 화면에는 비활성 관리자가 보이지 않습니다. 확인하려면 `비활성 포함 보기`를 켜세요. 다시 권한을 주려면 회색으로 표시된 행의 `재활성` 버튼을 누릅니다.

잠금 방지 규칙(운영진이 모두 로그인 못 하게 되는 사고를 막기 위한 안전장치):

- **본인 계정은 비활성화할 수 없습니다.** 현재 로그인한 본인 행에는 `본인` 배지가 표시되고 `비활성` 버튼이 항상 눌리지 않는 상태입니다. 본인 계정을 제외해야 한다면 다른 관리자에게 요청하세요.
- **활성 관리자가 1명뿐이면 그 관리자는 비활성화할 수 없습니다.** 마지막 남은 관리자를 비활성화하면 아무도 로그인할 수 없게 되므로, 항상 최소 1명의 활성 관리자가 유지되도록 서버에서 거부합니다.

### 전체 신청 목록

전체 신청을 확인하고 필요한 경우 취소합니다.

- 신청자
- 모임명
- 공간
- 날짜와 시간
- 신청 상태
- 취소 여부

최신 신청이 보이지 않으면 `새로고침`을 누릅니다.

### 관리자 차단 일정

특정 공간의 특정 날짜와 시간을 참가자가 예약하지 못하도록 막습니다.

예를 들어 외부 대관, 내부 행사, 점검, 촬영 일정이 있을 때 사용합니다.

### 공간 정보 관리

참가자 화면에 공간을 노출할지, 숨길지, 운영에서 제외할지 관리합니다.

또한 공간 설명, 특징, 정원, 관리자 메모를 수정할 수 있습니다.

## 예약 관리

신청 확인:

1. `전체 신청 목록`을 확인합니다.
2. 목록이 오래된 것 같으면 `새로고침`을 누릅니다.
3. 신청자, 모임명, 공간, 날짜를 확인합니다.
4. 상태 필터를 바꿔 취소된 신청도 확인할 수 있습니다.

상태 확인:

- 정상 신청은 기본 목록에 표시됩니다.
- 취소된 신청은 상태 필터에서 `취소됨`을 선택하면 확인할 수 있습니다.
- 참가자가 취소한 신청과 관리자가 취소한 신청 모두 취소됨 상태로 남습니다.

신청 취소:

1. 취소할 신청 행의 `취소` 버튼을 누릅니다.
2. 확인 모달에서 `신청 취소`를 누릅니다.
3. 성공 메시지를 확인합니다.
4. 참가자에게 `내 신청 확인/수정`에서 `새로고침`을 안내합니다.

주의사항:

- 관리자 취소 후 참가자 화면에서는 기본 목록에서 숨겨집니다.
- 참가자가 `취소된 신청 보기`를 켜면 취소된 신청을 확인할 수 있습니다.
- 취소는 삭제가 아니라 상태 변경입니다. 운영 기록을 남기기 위한 방식입니다.

## 차단 일정 관리

차단 일정은 참가자가 예약할 수 없는 시간을 설정하는 기능입니다.

저장:

1. 공간을 선택합니다.
2. 날짜를 선택합니다.
3. 시작시간과 종료시간을 입력합니다.
4. 사유를 입력합니다.
5. `차단 저장`을 누릅니다.

수정:

1. 기존 차단 일정에서 `수정`을 누릅니다.
2. 공간, 날짜, 시간, 사유를 바꿉니다.
3. `수정 저장`을 누릅니다.

해제:

1. 더 이상 막지 않아도 되는 차단 일정에서 `해제`를 누릅니다.
2. 참가자 예약 화면에 다시 예약 가능한 시간으로 반영되는지 확인합니다.

실제 운영 예시:

- 공간: 숨플레이스 A룸
- 날짜: 7월 17일
- 시간: 19:00~23:00
- 사유: 내부 행사

차단 일정 해제는 삭제가 아닙니다. 운영 이력을 남기기 위해 비활성 상태로 바뀝니다.

## 공간 관리

공간 관리에서는 참가자에게 보이는 공간 정보와 운영 상태를 관리합니다.

사용자 숨김:

- 참가자 화면에서 해당 공간을 숨깁니다.
- 데이터베이스에서 삭제되는 것은 아닙니다.
- 운영자가 다시 `사용자 노출`로 바꾸면 참가자 화면에 보입니다.

사용자 노출:

- 숨겨진 공간을 참가자 화면에 다시 표시합니다.
- 노출 전 운영시간과 기존 신청 영향을 확인하세요.

비활성:

- 공간을 운영에서 제외할 때 사용합니다.
- 단순히 참가자에게 잠시 숨기려는 목적이라면 `사용자 숨김`을 사용하세요.

설명 수정:

- 공간명, 상위 공간명, 정원, 대표 이미지 URL, 특징 태그, 설명을 수정할 수 있습니다.
- 저장 후 참가자 화면에서 설명이 자연스럽게 보이는지 확인합니다.

기존 공간 사진 교체:

- `수정 열기`를 누르면 `대표 이미지 URL` 아래에 `사진 파일 업로드`가 나타납니다.
- URL을 직접 입력하거나 컴퓨터의 파일을 선택할 수 있으며, 파일을 선택하면 URL 입력값 대신 업로드된 파일이 우선합니다.
- JPEG/PNG/WEBP/GIF만 가능하고, 5MB를 넘으면 거부됩니다.
- 실제 업로드는 `수정 저장` 버튼을 누르는 시점에 이뤄지며, 업로드에 실패하면 저장 자체가 중단되고 오류 메시지가 표시됩니다.
- 파일을 선택하지 않고 `수정 저장`을 누르면 기존과 동일하게 URL 입력값 그대로 저장됩니다.

기존 공간 운영시간 수정:

- `수정 열기` 안쪽 하단에 현재 운영시간 요약(예: `매일 09:00~21:00` 또는 요일별 줄 표시)이 보입니다.
- `운영시간 수정`을 누르면 시작·종료 시각(30분 단위)과 휴무 요일을 직접 지정하는 UI가 열립니다.
- 요일마다 다른 시간을 줄 수는 없고 공통 시간 + 휴무 요일 조합만 가능합니다. 티파티처럼 이미 요일별로 시간이 다른 공간을 열면 가장 흔한 시간대로 초기화되며 "요일별 상세 시간은 저장 시 공통 시간으로 통일됩니다" 경고가 표시됩니다. 이 상태에서 저장하면 요일별 차이는 사라집니다.
- 종료 시각이 시작 시각보다 늦지 않으면 저장할 수 없습니다.
- 운영시간 저장은 `수정 저장`(공간 정보 저장)과 별도 버튼(`운영시간 저장`)입니다. 두 저장은 서로 다른 RPC를 호출하므로, 공간 정보만 고치고 저장하면 운영시간은 그대로 유지되고 운영시간 RPC는 호출되지 않습니다.

사진 갤러리 관리(여러 장):

- `수정 열기` 안쪽, 운영시간 아래에 `사진 갤러리` 절이 있습니다. 이 절은 목업(Mock) 모드에서는 보이지 않고, 실제 Supabase 연결 상태에서만 나타납니다.
- 현재 등록된 활성 사진 썸네일이 등록순으로 나열되며, 대표 사진에는 `대표` 표시가 붙습니다.
- 사진 추가: `사진 파일 업로드`에서 파일을 선택한 뒤 `사진 추가`를 누르면 즉시 업로드되고 갤러리 맨 뒤에 새 사진으로 등록됩니다. 이 공간에 활성 사진이 하나도 없었다면 자동으로 대표 사진이 됩니다.
- 대표 지정: 대표가 아닌 사진의 `대표 지정`을 누르면 그 사진이 새 대표가 되고 기존 대표는 해제됩니다.
- 제거: 각 사진의 `제거`를 누르면 논리 삭제(비활성화)됩니다. 실제 데이터베이스 행은 남고 `is_active`만 꺼집니다. 대표 사진을 제거하면 남은 활성 사진 중 등록순이 가장 빠른 사진이 자동으로 새 대표가 됩니다.
- 참가자 화면(공간 카드, 예약 창, 사진 팝업)은 이 갤러리를 그대로 사용하므로, 저장 후 새로고침하면 참가자 화면에도 바로 반영됩니다.
- 사진 순서를 손으로 드래그해 바꾸는 기능은 없습니다. 등록순 + 대표 지정만으로 우선순위를 조정하세요.

관리자 메모:

- 참가자에게 보이지 않는 운영자용 메모입니다.
- 촬영 문의, 조율 가능 여부, 주의사항 등을 적어 둡니다.

새 공간 추가:

- `공간 정보 관리` 상단의 `새 공간 추가` 폼에서 신규 계약 공간을 등록합니다.
- 공간명, 상위 공간명, 카테고리(청년동 공간 / 생활밀착형 제휴공간), 정원, 사진 URL, 특징, 설명, 관리자 메모를 입력합니다.
- 사진은 `사진 URL`에 이미지 주소를 직접 입력하거나, 그 아래 `사진 파일 업로드`에서 컴퓨터의 파일을 선택할 수 있습니다.
  - JPEG/PNG/WEBP/GIF만 가능하고, 5MB를 넘으면 거부됩니다.
  - 파일을 선택하면 미리보기와 파일명이 표시되고, `선택 취소`로 다시 URL 입력값으로 되돌릴 수 있습니다.
  - 실제 업로드는 `공간 추가` 버튼을 누르는 시점에 이뤄지며, 파일을 선택한 경우 URL 입력값보다 업로드 파일이 우선합니다.
- 운영시간은 아래 프리셋 중 하나를 선택합니다.
  - `24시간`: 매일 00:00~24:00 운영.
  - `청년동 기본`: 매일 09:00~21:00 운영.
  - `칫챗 08:00-22:00`: 매일 08:00~22:00 운영.
  - `티파티 요일별`: 요일마다 운영시간이 다른 프리셋(월요일 휴무 포함).
  - `직접 설정`: 시작·종료 시각(30분 단위)과 휴무 요일을 직접 지정합니다. 요일마다 다른 시간을 줄 수는 없고, 공통 시간 + 휴무 요일 조합만 가능합니다. 종료 시각이 시작 시각보다 늦지 않으면 저장할 수 없습니다. 00:00~24:00로 설정해도 참가자 화면에는 09~22시로 제한되어 보이니 참고하세요.
- `활성`, `사용자 노출`, `관리자 허용 필요` 여부를 함께 지정할 수 있습니다.
- `공간 추가` 버튼을 누르면 저장 중 상태로 바뀌며, 저장이 끝나면 목록에 바로 추가됩니다. 실패하면 입력 폼 아래에 오류 메시지가 표시되며 입력값은 유지됩니다.
- 이미 존재하는 공간 ID와 겹치면 저장되지 않습니다(자동 생성되는 ID라 일반적으로는 발생하지 않습니다).

주의사항:

- `사용자 숨김`은 참가자 화면만 숨기는 기능입니다.
- 공간 데이터 삭제가 아닙니다.
- 공간 물리 삭제는 관리자 화면에서 제공하지 않습니다.
- 사진 갤러리 제거도 물리 삭제가 아니라 논리 삭제(`is_active = false`)입니다.

## 운영 종료 후

운영 종료 후 또는 운영 시작 전 테스트 데이터를 정리해야 할 때 아래를 확인합니다.

확인할 데이터:

- `meetings`: 예약 모임 데이터
- `sessions`: 예약 시간 데이터

운영 시작일에 신청 목록이 비어 있어야 한다면 두 값이 0인지 확인합니다.

초기화 절차:

1. 관리자 화면에서 현재 신청 목록을 확인합니다.
2. 운영 기록을 보존해야 하는 신청이 없는지 확인합니다.
3. 개발 담당자 또는 SQL 실행 담당자에게 초기화를 요청합니다.
4. [supabase/manual-sql](./supabase/manual-sql)의 `03_clear_test_reservations.sql` 또는 `05_emergency_reset.sql`을 사용합니다.
5. `04_verify_counts.sql`로 `meetings = 0`, `sessions = 0`을 확인합니다.

운영자가 SQL에 익숙하지 않다면 직접 실행하지 마세요. 잘못 실행하면 운영 데이터에 영향을 줄 수 있습니다.

## 자주 발생하는 문제

신청이 안 보일 때:

1. 관리자 화면에서 `새로고침`을 누릅니다.
2. 상태 필터가 `취소됨` 또는 다른 값으로 되어 있지 않은지 확인합니다.
3. 참가자가 실제로 신청 완료 메시지를 봤는지 확인합니다.

관리자 취소 후 참가자 화면이 다를 때:

1. 참가자에게 `내 신청 확인/수정`의 `새로고침`을 안내합니다.
2. `취소된 신청 보기`를 켜서 취소 상태를 확인하게 합니다.

차단 일정이 적용되지 않는 것 같을 때:

1. 공간, 날짜, 시작시간, 종료시간이 맞는지 확인합니다.
2. 차단 일정이 해제 상태가 아닌지 확인합니다.
3. 참가자 화면에서 날짜를 다시 선택해 봅니다.

공간이 참가자 화면에 보이지 않을 때:

1. `공간 정보 관리`에서 `사용자 노출` 상태인지 확인합니다.
2. `활성` 상태인지 확인합니다.
3. 숨김 또는 비활성 상태라면 운영 목적에 맞게 변경합니다.

더 자세한 문제 확인 순서는 [TROUBLESHOOTING.md](./TROUBLESHOOTING.md)를 확인하세요.
